# Desktop redesign action inventory

This inventory covers the controls introduced or materially repositioned by the desktop redesign. Existing forms loaded inside lazy sections retain their original handlers and validation. Repeated navigation links use the same delegated shell handler and are grouped below.

| Surface / visible control | Template selector | Handler or destination | Write / confirmation behavior | User feedback and access | Verification |
| --- | --- | --- | --- | --- | --- |
| Sidebar links, top-bar Import, summary cards, rail links | `[data-app-nav-link]` | `activateAppShellNavLink()` delegates to the declared `data-app-nav-action`, page target, settings section, or lazy section | Read-only navigation | Active state, hash update, lazy loading status; signed-in and guest destinations retain their existing guards | Shell design-system tests + browser QA |
| Collapse / expand sidebar | `[data-app-sidebar-collapse]` | `setAppSidebarCollapsed()` | Saves only the local UI preference in `localStorage` | `aria-label`, `aria-pressed`, title, and glyph update immediately | Meal Planner template test + browser interaction QA |
| Global search and Ctrl/Cmd+K | `[data-app-global-search]` | `submitAppGlobalSearch()` and the shell keydown handler | Read-only navigation | Inline status for empty/no-match/matched section | Existing shell navigation tests + browser QA |
| Home recipe cards | `[data-recipe-url]` | `jumpToRecipeViewRecipe()` | Read-only navigation | Opens the selected persisted recipe in the existing recipe view | Existing recipe-view tests + browser QA |
| Home Add a meal / View planner | `openMealPlannerPageAndDialog()` or `data-app-page-target="mealPlannerPage"` | Opens the weekly planner and, for Add a meal, its modal | No write until the modal form is submitted | Modal focus management and empty-state copy | Meal Planner tests + browser QA |

## Controls intentionally represented as labels

The single current collection states (for example, **All Recipes**, **All Cookbooks**, **All Menus**, and **Current List**) are non-interactive labels, not buttons. Unsupported mockup-only tabs, favorites, price histories, usage totals, and notification events are not rendered as functioning data.
